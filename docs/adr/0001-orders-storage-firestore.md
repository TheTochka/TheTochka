# ADR 0001: хранение заказов в Firestore

## Статус

Принято (MVP).

## Контекст

Нужен записанный заказ со статусом и переходами до появления отдельной OLTP-БД.

## Решение

- Коллекция **`orders`** в Firestore; чтение/запись **только через Firebase Admin SDK на VPS**.
- Клиентское приложение **не пишет** в `orders` (в `firestore.rules` запрет; фактически весь клиентский доступ к Firestore уже закрыт).

## Последствия

- Идемпотентность webhook и провижнинг реализуются на сервере (транзакции, отдельная коллекция `webhook_events`).
- При смене хранилища заказов потребуется миграция данных и обновление VPS API.
