# Юридическая концепция TOCHKA (RU)

Последнее обновление: 15 апреля 2026

## 1) Модель сервиса

TOCHKA — **информационный агрегатор (каталог)** провайдеров подключения.

TOCHKA:
- не продает пользователю услуги подключения;
- не принимает платежи за услуги провайдера в приложении;
- не является стороной договора между пользователем и провайдером;
- не гарантирует качество, доступность, стоимость и результат услуг провайдера.

Функция приложения — предоставить пользователю:
- структурированную информацию о провайдерах;
- возможность отправить провайдеру запрос на предложение;
- каналы связи с поддержкой приложения по техническим вопросам.

## 2) Что важно для сторов

### Apple
- Если в приложении продаются цифровые услуги/контент, применяется IAP.
- Если нет продаж внутри приложения, тексты и интерфейс не должны создавать впечатление покупки внутри приложения.
- При наличии аккаунта нужен путь удаления аккаунта и данных.

Ссылки:
- https://developer.apple.com/app-store/review/guidelines/
- https://developer.apple.com/support/offering-account-deletion-in-your-app/

### Google Play
- Внутри приложения нельзя вводить пользователя в заблуждение о функциональности.
- Если внутри нет продаж, тексты не должны содержать утверждений о покупке/подписке в приложении.
- При наличии аккаунта требуется удаление аккаунта и связанных данных (in-app путь + web путь).

Ссылки:
- https://support.google.com/googleplay/android-developer/answer/9888077
- https://support.google.com/googleplay/android-developer/answer/9858738
- https://support.google.com/googleplay/android-developer/answer/10281818
- https://support.google.com/googleplay/android-developer/answer/13327111

## 3) Единые формулировки для всех страниц

Рекомендуемые обязательные формулировки:

1. "TOCHKA является информационным агрегатором провайдеров подключения."
2. "TOCHKA не оказывает услуги подключения и не является стороной договора между пользователем и провайдером."
3. "TOCHKA не принимает оплату за услуги провайдера в приложении."
4. "Условия, цены, доступность и качество услуг определяются и предоставляются соответствующим провайдером."
5. "Запрос пользователя передается провайдеру; решение о направлении предложения принимает провайдер."

## 4) Практическое правило контента

Нельзя использовать на сайте и в приложении формулировки:
- "оформите подписку в приложении";
- "оплата через наш шлюз";
- "возврат средств за подписку в приложении";
- "мы предоставляем подключение".

Нужно использовать:
- "отправить запрос провайдеру";
- "получить предложение от провайдера";
- "сравнить условия провайдеров";
- "каталог/агрегатор информации".
